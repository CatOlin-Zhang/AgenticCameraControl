# Camera Control Skill

Intelligently discover, connect, and control ONVIF/RTSP network cameras and local USB cameras through natural language or slash commands. An embedded LLM (Qwen2.5-0.5B local or Ollama) parses user intent into structured commands with automatic parameter filling.

## Quick Start

```bash
# Launch CLI
python -m phase3.main

# Or specify a config file
python -m phase3.main --config path/to/config.yaml
```

On startup the system loads `config.yaml`, initializes the camera manager and LLM client, then enters the interactive CLI prompt (`📷 >`).

## Core Workflow

### Step 1 — Discover cameras

The system supports two discovery modes:

- **Passive WS-Discovery**: listen for Hello heartbeat packets on multicast `239.255.255.250:3702`. Cameras broadcast Hello when they come online.
- **Active Probe**: send a WS-Discovery Probe to trigger ProbeMatch responses from already-online cameras.
- **USB enumeration**: scan device indices 0–9 via OpenCV.

```
📷 > /discover_net 15       # passive listen for 15s + active probe
📷 > /discover               # scan USB cameras
📷 > /listen                 # background continuous heartbeat listener
```

Or via natural language: `"扫描局域网摄像头"`, `"自动设置"`.

### Step 2 — Set password and connect

Discovered cameras are auto-registered as `discovered_<ip>`. Set the password before connecting:

```
📷 > /password mySecret123   # syncs to all ONVIF cameras
📷 > /connect                 # connects the default camera
📷 > /probe_all               # probe all registered cameras, summarize results
```

The system tries ONVIF first; if unavailable it falls back to RTSP-direct mode, auto-trying common stream paths (Hikvision, Dahua, XM, etc.).

### Step 3 — Stream and preview

```
📷 > /stream                  # USB: open preview window; ONVIF: show RTSP URL + open preview
📷 > /preview                 # open live preview (press 'q' to close, 's' to snapshot)
📷 > /snapshot                # capture a frame, save to snapshots/
```

Press `q` in the preview window to close; press `s` to save a snapshot.

### Step 4 — PTZ control (ONVIF only)

```
📷 > /ptz up 0.5             # move up at speed 0.5 (auto-stops after 1s)
📷 > /ptz left               # move left at default speed
📷 > /ptz_stop               # immediate stop
📷 > /ptz_zoom in            # zoom in (auto-stops after 1.5s)
📷 > /presets                # list PTZ presets
```

PTZ requires ONVIF connection. USB and RTSP-only cameras do not support PTZ.

## Natural Language Commands

The LLM parses user input into one of these commands. No parameters needed — the system auto-fills them.

| Command | Example triggers |
|---------|-----------------|
| `discover_network` | "扫描摄像头", "find cameras on network" |
| `connect_camera` | "连接摄像头", "connect the camera" |
| `watch_camera` | "看看画面", "show me the feed" |
| `get_stream` | "拉流", "get the stream URL" |
| `take_photo` | "截个图", "take a photo" |
| `open_preview` | "打开预览", "open preview" |
| `get_status` | "查看状态", "camera status" |
| `list_cameras` | "列出设备", "what cameras are there" |
| `set_password` | "密码是 abc123", "set password" |
| `auto_setup` | "自动设置", "set up everything" |

**Compound commands** auto-orchestrate multiple steps:

- `auto_setup` → discover → register → connect → stream
- `watch_camera` → find available → connect → stream → preview
- `take_photo` → find available → connect → capture → save

## LLM Intent Parsing

The system uses a **two-pass** architecture to handle small models reliably:

1. **Pass 1 — Streaming chat**: the LLM generates a conversational reply while the user watches the real-time output. If the reply contains a JSON command, it is extracted.
2. **Pass 2 — Forced extraction**: if Pass 1 yields no command, a dedicated minimal prompt forces the model to output only `{"command":"name"}`. This isolates intent classification from conversation context.

Parameters are **never** generated by the LLM (except `set_password`). All parameters are auto-filled by code based on current system state.

## Edge Cases & Troubleshooting

| Issue | Cause | Fix |
|-------|-------|-----|
| No cameras discovered | Multicast blocked or cameras offline | Restart cameras; ensure same LAN; extend listen timeout |
| ONVIF connect fails | Wrong port or missing credentials | Check XAddrs port; set password with `/password` |
| RTSP fallback fails | Non-standard stream path | Add correct `rtsp_path` in config.yaml |
| PTZ not working | RTSP-only connection (no ONVIF) | Ensure ONVIF connection succeeds; check camera PTZ support |
| LLM returns no command | Model too small for context | System auto-falls back to Pass 2 extraction; simplify input |
| Preview window encoding error | Windows console UTF-8 issue | System auto-handles; re-enter input if garbled |

  backend: local
